## Ansible — Handler to Restart Nginx

---
## Name of the play 
- name: Manage Nginx
  hosts: all ## run this play on all host
  become: yes ## Execute tasks with sudo/root privileges.



  tasks: ## START LIST OF TASK TO EXECUTE
    - name: Update config file  ## nAME OF TASK
      copy: ## USE ANSIBLE COPY MODULE
        ## COPY DATA FROM ANSIBLE CONTROL NODE TO TARGET SERVER
        src: index.html 
        dest: /usr/share/nginx/html/index.html
      notify: Restart Nginx 
      ##Triggers a handler named Restart Nginx.

  handlers:
    - name: Restart Nginx
      service:
        name: nginx
        state: restarted

## 🔹 Kubernetes — Resource Limits

apiVersion: v1 ## kubernetes api version
kind: Pod # create a pod
metadata: # info about pod
  name: pathnex-pravin-limit-pod ## name of pod
spec:  # desiresd state of pod
  containers:  ## list of container inside pod
    - name: app
      image: nginx ## use of nginx image for docker hub
      resources: ## control cpu and memory allocation
        limits: 
          cpu:    "256Mi"
          memory: "500Mi"



apiVersion: v1
kind: Pod
metadata: 
  name: Pathnex-Limited-Pod

  - name: app

# Day 05 — Environment Variables

## 🔹 Jenkins Pipeline — Use Environment Variables
You will learn how to **use environment variables in Jenkins pipelines**.


pipeline {
  ## Pipeline can run on any available Jenkins agent/node
  agent any
  
  ## Defines environment variables.
  ## Variables are available to all stages in the pipeline
  environment {
    INSTITUTE_NAME = "Pathnex"
  }
  ## Contains all stages.
  ## Stages execute sequentially.
  stages {
      stage('Print'){
          steps {
              sh 'echo "Welcome to $INSTITUTE_NAME DevOps Training"'    ## run shell cmd on linux
          }
        }
        stage ('Build') {
          steps {
            sh 'mvn clean package -Dinstitute.name=$INSTITUE_NAME'  ## clean previously build artifact
          }
        }
    }
}
